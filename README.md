# 群聊内容审核插件

基于大语言模型（LLM）的 AstrBot 群聊消息自动审核插件，可定时爬取指定群聊的最新消息，通过 AI 工具调用判定是否含有违法、色情、辱骂内容，并自动撤回违规消息。  
支持自动将拥有管理员/群主权限的群加入监控名单，无需手动添加。

---

## ✨ 功能特性

- **AI 工具强制调用** —— 将审核逻辑封装为一个 LLM 必须调用的 `report_violations` 工具，AI 在发现违规时返回具体的违规消息编号，避免误判。
- **定时自动审核** —— 可配置审核间隔和每次爬取的消息数量，完全无人值守。
- **违规消息撤回** —— 根据 AI 返回的编号精准撤回对应消息（需机器人拥有群管理权限）。
- **自动添加监控群** —— 可选开启，插件加载时扫描所有已加入的群，将机器人是群主或管理员的群自动加入审核名单。
- **高度可定制** —— 支持自定义审核提示词、LLM 提供商、监控群列表等，通过 WebUI 可视化配置。

## 🚀 安装

1. 确保 AstrBot 已正确安装且正常运行，**aiocqhttp 平台已连接**。
2. 直接在插件市场搜索astrbot_plugin_group_content_moderation，点击安装
3. 在 AstrBot WebUI → AstrBot 插件 → 找到本插件，点击重载或启用。
4. 进入插件配置页，根据需求设置审核参数（见下文）。
5. 重启或重载插件使配置生效。

---

## ⚙️ 配置说明

所有配置均可在 WebUI 中直接修改，无需编辑代码。  
配置项如下：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `group_ids` | list | `[]` | 要监控的群号列表，例如 `[123456, 789012]`。**可通过 WebUI 直接编辑 JSON 数组**。 |
| `auto_add_groups` | bool | `false` | 是否在插件加载时自动扫描所有已加入的群，将机器人拥有管理员或群主权限的群加入 `group_ids`。新加的群号会保存到配置中。 |
| `fetch_interval_minutes` | int | `10` | 审核间隔时间（分钟），即每隔多少分钟执行一次全群审核。 |
| `fetch_count` | int | `50` | 每次从每个群获取的最新消息条数。受协议端限制，实际可获取的数量可能小于此值。 |
| `moderation_prompt` | text | 空（使用内置提示词） | 发送给 LLM 的 **系统提示词**，用于指导 AI 如何进行审核和调用工具。留空则使用内置提示词。可自由定制，例如约束审核范围、违规判定标准等。 |
| `chat_provider` | string | 空（自动选择第一个可用） | 用于审核的 **LLM 提供商**。点击右侧按钮可在已配置的提供商中选择。若不指定，插件会自动使用第一个已启用的 LLM 提供商。 |

### 内置提示词（当 `moderation_prompt` 为空时）
> 你是一个群聊管理员。请检查以下消息，判断是否包含违法、色情或辱骂内容。你必须使用 report_violations 工具上报结果：将违规消息的编号放入 violation_ids 数组，如果没有违规，就传入空数组 []。

## 📘 使用方法

1. 在配置中填入要监控的群号，或者开启 `auto_add_groups` 让插件自动添加有权限的群。
2. 确认选择的 LLM 提供商可用，且拥有正常的对话能力。
3. 设置合适的审核间隔和抓取数量，建议默认值起步。
4. 确保机器人在目标群中 **拥有管理员或群主权限**，否则无法撤回消息。
5. 重载插件后，插件会立即执行一次自动添加群（若开启），并启动定时审核任务。

运行日志中会输出审核结果及撤回信息：
```
[INFO] 已撤回违规消息 -123456（群 789012，编号 3）
[INFO] 已撤回违规消息 -123457（群 789012，编号 5）
```

## 🧠 工作原理

1. **定时触发**  
   插件根据 `fetch_interval_minutes` 启动一个异步循环，每隔指定分钟数执行一次审核流程。

2. **拉取消息**  
   对监控列表中的每个群，通过 OneBot 协议调用 `get_group_msg_history` 获取最近 `fetch_count` 条消息，并过滤掉无文本内容的消息（如图片、语音），为每条文本消息分配从 1 开始的编号。

3. **构造 Prompt 并调用 LLM**  
   将消息列表以 “编号. 消息内容” 的格式发送给 LLM，同时携带系统提示词。  
   LLM **必须调用** 内置的 `report_violations` 工具，该工具的参数为 `violation_ids`（整数数组）。

4. **工具调用返回违规编号**  
   - 若存在违规，AI 会在工具参数中返回具体编号，例如 `[1, 4, 5]`。  
   - 若无违规，AI 传入空数组 `[]`。  

5. **撤回消息**  
   插件解析工具调用的结果，根据编号找到真实的 `message_id`，调用 `delete_msg` API 撤回消息，并记录日志。

---

## ⚠️ 注意事项

- **撤回权限**：机器人必须在目标群中为 **管理员或群主**，否则 `delete_msg` 会失败，日志中会记录错误，但不影响其他群审核。
- **消息历史接口限制**：不同协议端对 `get_group_msg_history` 的支持程度不同，可能只能获取最近几十条消息，或需要特定权限。请参考所用协议端的文档。
- **AI 调用稳定性**：LLM 可能未按预期调用工具，或返回格式错误。插件已添加空值保护，若工具调用参数缺失则不会撤回任何消息。
- **配置保存**：修改配置后建议重载插件，特别是 `group_ids` 的自动添加功能仅在插件加载时执行一次。
- **多群并发审核**：目前是顺序处理每个群，若监控群数较多且审核间隔过短，可能导致 LLM 并发请求排队，可根据资源调整间隔。

## ❓ 常见问题

**Q：为什么消息撤不回？**  
A：请检查机器人是否在该群拥有管理员/群主权限，以及协议端是否支持 `delete_msg` API（多数 OneBot 实现均支持）。

**Q：AI 总是判定为无违规，但实际有违规内容怎么办？**  
A：可以尝试修改 `moderation_prompt`，给出更明确的判定标准，或更换更强的 LLM 模型。

**Q：自动添加群为什么不生效？**  
A：自动添加仅在插件**加载**时执行一次（包括启动和手动重载）。若之后新加入群聊，需要再次重载插件或手动添加群号。

**Q：可以不使用 AI，直接用关键词过滤吗？**  
A：本插件设计为 **基于 LLM 的语义级审核**，若需关键词过滤可另行开发或使用其他插件。
